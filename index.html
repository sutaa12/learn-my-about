GitHub Pages の静的サイト見本

ブランチ別サイトを、迷わず見つけられる入口をつくる。

このページは、ブランチごとに用意した index.html を GitHub Action で集約し、 ひとつのトップページから辿れるようにするための見本ページです。知識がない人でも流れを追いやすいように、 何が起きているかを順番に噛み砕いて紹介しています。

どんなときに便利？ 実験的な案・派生ページ・ブランチごとの比較を、リンク探しで迷わず確認できます。
description.md の役割 各ブランチの説明文をトップページに載せられるので、開く前に内容を判断しやすくなります。
スマホ対応 カードや図解の幅が自動調整されるので、縦長の画面でも読みやすく保てます。

しくみを 3 ステップで図解

ライトモード前提 カラフル モバイル対応
1. ブランチにページを置く

それぞれのブランチのルートに index.html を置きます。説明文を加えたいときは description.md も一緒に置きます。

2. GitHub Action がまとめる

ワークフローが全ブランチを確認し、見つかった静的ファイルを GitHub Pages 用の 1 つの成果物に整理します。

3. トップページから移動する

ブランチ名の階層を見やすく並べたトップページから、見たいブランチの index.html をそのまま開けます。

このページで意識したこと

元情報を省略しすぎない
  • どのファイルを置けば公開対象になるかを明示
  • GitHub Pages 向けの静的サイト前提と説明
  • 階層ブランチや description.md の扱いも紹介
見た目で流れが分かる
  • 大きな見出しで目的を先に伝える
  • カードと図解で手順を視覚化
  • 色の役割を絞って情報を整理
GitHub Pages で扱いやすい
  • HTML と CSS だけで表示できる構成
  • 相対パス中心で静的配信に馴染む
  • 外部依存を増やさずに軽量に公開
